# 部署指南

## 环境要求

- Node.js >= 18
- npm >= 9

## 生产环境部署

### 1. 构建前端

```bash
cd frontend
npm install
npm run build
```

构建产物在 `frontend/dist` 目录

### 2. 配置后端

```bash
cd backend
npm install
```

创建 `.env` 文件：

```env
PORT=3001
JWT_SECRET=your-secret-key
```

### 3. 初始化数据库

```bash
# 后端首次运行会自动创建数据库
# 如需导入种子数据：
sqlite3 database/resume_template.db < database/seed.sql
```

### 4. 启动后端服务

```bash
cd backend
npm start
```

### 5. 配置反向代理（推荐Nginx）

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /path/to/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # API代理
    location /api {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

## Docker部署（可选）

### Dockerfile (后端)

```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY backend/package*.json ./
RUN npm install
COPY backend/ .
EXPOSE 3001
CMD ["npm", "start"]
```

### 启动命令

```bash
# 启动后端
docker run -d -p 3001:3001 \
  -e PORT=3001 \
  -e JWT_SECRET=your-secret \
  resume-backend

# Nginx处理前端静态文件和API代理
```

## 目录结构

部署时确保以下目录结构：

```
/app
├── frontend/
│   └── dist/           # 构建产物
├── backend/
│   └── src/
├── database/
│   └── resume_template.db  # SQLite数据库
└── .env
```

## 注意事项

1. 生产环境务必修改 `JWT_SECRET`
2. 数据库文件需要定期备份
3. 建议使用PM2管理后端进程
4. 前端构建后无需Node.js环境

## PM2部署后端

```bash
npm install -g pm2
cd backend
pm2 start src/index.js --name resume-api
pm2 save
pm2 startup
```

# 简历模板站 - 运行指南

## 🔧 环境要求

- **Node.js** ≥ 18.0.0（推荐 20.x 或 22.x LTS 版本）
- **npm**（随 Node.js 自带，无需单独安装）
- **Windows / macOS / Linux** 均可

### 检查当前环境

打开 **PowerShell** 或 **命令提示符 (CMD)**，输入：

```
node -v
npm -v
```

- 如果你看到类似 `v20.11.0` 的版本号 → 已安装，跳过下一步
- 如果你看到 `'node' 不是内部或外部命令` → 按下面步骤安装

1. 打开官网：**https://nodejs.org/zh-cn**

## 🔌 API 接口文档（后端端口 3001）

所有接口返回格式统一为 `{ code, message, data }`，其中：
- `code: 0` 成功
- `code: HTTP 状态码` 失败（如 400/401/404/500）

### 用户接口 `/api/users`
- `POST /register` — 注册（body: `{ username, email, password }`）
- `POST /login` — 登录（body: `{ username, email, password }`）
- `GET /me` — 获取当前用户（header: `Authorization: Bearer <token>`）
- `PUT /me/pro` — 升级 Pro 会员（模拟）

### 模板接口 `/api/templates`
- `GET /` — 模板列表，支持 `?category=&language=&is_pro=&search=&page=&limit=`
- `GET /featured` — 精选模板
- `GET /:id` — 模板详情
- `POST /:id/download` — 增加下载计数

### 简历接口 `/api/resumes`（需登录）
- `GET /` — 我的简历列表
- `GET /:id` — 单份简历详情
- `POST /` — 保存简历草稿
- `PUT /:id` — 更新简历
- `DELETE /:id` — 删除简历

### 收藏接口 `/api/favorites`（需登录）
- `GET /` — 我的收藏列表
- `POST /` — 新增收藏（body: `{ template_id }`）
- `DELETE /:template_id` — 取消收藏
- `GET /check/:template_id` — 检查是否已收藏

**测试示例**（在 PowerShell 中运行）：

```powershell
# 注册新用户
curl -X POST http://localhost:3001/api/users/register -H "Content-Type: application/json" -d '{"username":"testuser","email":"test@test.com","password":"123456"}'

# 查看模板列表
curl http://localhost:3001/api/templates
```

## ⚙️ 可选配置（.env 文件）

如需自定义端口或 JWT 密钥，在对应目录新建 `.env` 文件：

**后端 `backend/.env`**：
```
PORT=3001
JWT_SECRET=你自定义的密钥字符串（随便写，越长越安全）
CORS_ORIGIN=*
BODY_LIMIT=2mb
NODE_ENV=development
```

**前端 `frontend/.env`**（一般不需要，默认端口已在 vite.config.js 中设置）：
```
VITE_API_BASE_URL=/api
```

## 🐛 常见问题排查

### 问题 1：`npm install` 卡住或非常慢
**原因**：npm 默认从国外服务器下载包。
**解决**：切换到国内镜像：
```powershell
npm config set registry https://registry.npmmirror.com
```
验证：`npm config get registry` 应显示 `https://registry.npmmirror.com`

### 问题 2：端口被占用（报错 `EADDRINUSE`）
**解决**：
- 关闭正在使用该端口的其他程序，或
- 修改 `backend/package.json` 的端口号（把 3001 改成 3002 等）
- 修改 `frontend/vite.config.js` 中的 proxy 目标也要对应改

### 问题 3：前端页面打开了但数据加载失败
**排查步骤**：
1. 确认后端服务是否在运行（步骤 2 的那个窗口不能关）
2. 浏览器按 F12 打开开发者工具 → 看 Console（控制台）和 Network（网络）面板
3. 访问 `http://localhost:3001/api/health` 看是否返回正常 JSON
4. 如果健康检查正常但前端失败 → 检查前端 proxy 配置
5. 如果健康检查失败 → 看后端控制台报错信息

### 问题 4：浏览器打开 `localhost:3011` 显示空白
**解决**：
1. 按 F12 打开开发者工具 → Console
2. 看是否有报错（红色文字）
3. 将报错信息截图发给我，我来帮你修复

### 问题 5：PDF 导出是空白
**解决**：
1. 确保你在简历生成器页面填写了至少姓名等内容
2. 如果预览正常但 PDF 空白 → 是 html2canvas 跨域问题（纯前端一般不会触发）
3. 刷新页面重试
